# Mock vs MockitoBean

## Когда достаточно обычного Mock

Mock создает объект, у которого все методы по умолчанию возвращают `null` (или значения по умолчанию для примитивов).

Если цель - написать быстрый тест без необходимости поднимать Spring-контекст, то вполне можно обойтись обычным `Mock`.

## Когда использовать MockitoBean

Если цель - поднять Spring-контекст и заменить стандартный бин на свою реализацию для тестирования, то стоит использовать `MockitoBean`.

Частый пример - тестирование контроллеров, когда нужно подменить сервисный слой мок-объектом и проверить только логику контроллера.

## Кратко

- Нужен быстрый тест без Spring → используем **Mock**.
- Нужен Spring-контекст и замена бина → используем **MockitoBean**.
