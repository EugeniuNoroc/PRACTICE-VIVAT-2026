## dirty-checking в Hibernate
### EntityManager
`Hibernate` захватывает `Connection` во время первого обращения к БД или открытия транзакции, persisted-сущности всегда имеют `id`, каждый экземпляр `EM` располагает доступом в кэш первого уровня, в котором объекты сравниваются по ссылке.
### dirty-checking
Соответственно механизм, который выполняется во время `flush`. Цель - найти сущности, которые изменились и обновить их. Для этого он обращается к оригинальной копии полей объекта которую создает `Hibernate` и сравнивает с актуальным объектом, точнее с его полями.  
Dirty-checking работает только для `Persistent` объектов, `Detached` объекты не отслеживаются - изменения не попадут в БД, а `Transient` объекты Hibernate вообще не видит.
