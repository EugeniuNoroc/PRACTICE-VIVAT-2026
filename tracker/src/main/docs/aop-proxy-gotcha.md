## AOP подводный камень

Spring реализует `@Transactional` через AOP прокси - при вызове метода извне
Spring перехватывает вызов через прокси-обертку и открывает транзакцию.

Если метод внутри класса вызывает другой метод того же класса - вызов идёт
напрямую через this, минуя прокси. Прокси не перехватывает вызов, поэтому
`@Transactional` на вызываемом методе игнорируется.

Следствие: если внутренний метод помечен `REQUIRES_NEW` - новая транзакция
не создается. Он выполняется в той же транзакции, что и вызывающий метод.
Если внутренний метод падает - `rollback-only` метка распространяется на всю 
транзакцию включая вызывающий метод.

Решения:
1) `Self-injection` - вместо `this.method()` инжектируешь сам себя как Spring-бин и вызываешь через него. Тогда вызов идёт через прокси:
```java
@Service
public class TaskService {
    @Lazy @Autowired
    private TaskService self; // Spring инжектирует прокси-версию себя
    
    public void createWithAudit() {
        self.logAudit(); // через прокси - @Transactional работает
    }
}
```
2) `TransactionTemplate` - программный способ открыть транзакцию вручную без аннотации:
```java
transactionTemplate.execute(status -> {
    // код внутри транзакции
    return result;
});
```
3) Вынести метод в отдельный Spring-компонент
4) `@Transactional` на самом верхнем методе

Та же проблема с `@Async` - internal call не создаёт новый поток.

------------

| Propagation    | Поведение                                              |
|----------------|--------------------------------------------------------|
| `REQUIRED`     | Присоединяется к существующей или создаёт новую        |
| `REQUIRES_NEW` | Всегда новая, старая приостанавливается                |
| `NESTED`       | Savepoint внутри текущей (как W1 savepoint в JDBC)     |
| `SUPPORTS `    | Использует если есть, без транзакции если нет          |
| `NOT_SUPPORTED`| Приостанавливает текущую, выполняется без транзакции   |
| `MANDATORY`    | Требует существующую, иначе исключение                 |
| `NEVER`        | Запрещает транзакцию, иначе исключение                 |
