### persistance-context, first-level cache, flush modes - этаче
- `persistance-context` - это что-то типа рабочей области где Hibernate хранит managed-объекты и их снапшоты, с которыми при коммите сравнивает конечный объект для того сделать с ним какие-то действия в БД.
- `first-level cache` - кэш внутри persistence-context, который хранит уже загруженные объекты. Нужен для того чтобы быстро получать доступ к объектам, а не каждый раз делать запрос в бд, так как это дорого и долго.
- `flush` - он проходит по всем managed-объектам в persistence-context, сравнивает их со своим сохраненным снапшотом и генерирует нужный SQL. Делается это для того, чтобы Hibernate мог понять какой SQL ему делать, например `INSERT` или `UPDATE`.  
`Flush modes` определяют когда происходит flush: `AUTO` (по дефолту) - делает перед каждым JPQL/SQL запросе и коммите, `COMMIT` - только при коммите.
